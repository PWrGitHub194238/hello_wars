General information about Arena.
  • Arena is a environment which is designed for playing games beetween many players. Arena uses settings information from ArenaConfiguration.xml and based on this data runs proper game which is played according selected elimination rules.
  • Bot it is web application which is able to response for two calls:
    • Info() - returns base information about bot and his owner.
      • <string> Name - your bot name
      • <string> AvatarUrl - path to your avatar
      • <string> GameType - type of game that your bot is able to play
      • <string> Description
    • PerformNextMove(TInfo info) returns TMove class. Bot is called whenever Arena decide that it is his turn to do the move. Arena shares with bot TInfo class which contains information about situation in duel (usually position of obsticles and oponents position). After running some highly advanced, high tech algorithm bot returns TMove to Arena.
  • Elimination Plugin- description
  • Game Plugin - description
  • ArenaConfiguration.xml- XML configuration file used by Arena, contains Bots links and some settings to Game and Elimination Controls.
    • EliminationConfiguration
      • Type - runs selected elimination.
    • GameConfiguration
      • Type - runs selected gamr.
      • NextMoveDelay - time in miliseconds between each bot move
      • NextMatchDelay - time between next matches
    • BotUrls - bot adres in localhost or in local network (if bot is set) ex.
      • <url>http://localhost:53886/TicTacToeBot/</url>
      • <url>http://67221-tpe540:81/tankblasterbotveronica/</url>
Bot written in .Net
  • Setup sample bot on IIS server
    • Open some bot project from "dotNet" folder, rebuild it or restore packages for solution and then rebuild. Run solution. Bot should be visible in IIS server.
    • open iis and add bot example from "dotNet" folder project - right click on "Sites" -> "Add Web Site"
    • change Port number to some unused port
    • type and remember web site name it will be application pool and you will need it.
    • choose path to application folder
    • click OK.
    • right clic on bot web site then "Edit Permissions" and "Security" bookmark
    • click "Edit" then by clicking "Add" add the new user with permissions for IIS .... something, type IIS APPPOOL\{application pool} in text box, change Location for your local host location ie. computer name. Click Ok.
    • Choosing your user allow to has Full controll, click OK and go back to IIS.
    • run sample bot if not running
    • make sure that Windows Features -> Internet Information Services -> WWW services -> Asp.Net 4.5 is checked.
    • check bot in web browser ex. "http://localhost:53886/antwarsbot/info" or http://67221-tpe540:81/tankblasterbotveronica/info or http://192.168.104.226:81/tankblasterbotveronica/info
  • If you wish that your bot should be visible from outside. Configure your firewall.
    • open Windows Firewall -> Advanced Settings -> Inbound Rules - add exception for port on which your bot exist.
    • configure other firewall if installed.
    • your bot adress is for example: http://192.168.104.226:81/tankblasterbotveronica/info
  • Modify ArenaConfiguration file:
    • change game type you wish to play. Note if you selected bot which could play TicTacToe game, you have to play TicTacToe game.
    • change elimination type name ex. RoundRobin, LadderTournament, so far only this two are available.
    • modify url to bot on your computer, you should put a few identhical links so that simulate few competitors.
  • While solution with bot is running or bot is set up on IIS run Arena. Arena should automathicaly connect with bots. Now press play to see the duel...
